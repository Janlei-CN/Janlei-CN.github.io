接口隔离原则

 实现类只实现需要实现接口的方法，若实现类实现接口中无需用到的方法，但是也强制重写了实现方法，这就违背了接口的隔离性。
 简单来说就是，A实现类需要实现B接口oper1与oper2方法，但是B接口在定义的时候却包含了oper1与oper2以外的方法oper3。这样A在实现B接口的时候，必须实现B接口的所有方法，但是A除了oper1与oper2以外的方法是不需要的，这样强制重写了B的oper3方法。这就违背了接口的隔离原则，也违背了类设计的单一功能职责。

improve

 解决上述违背接口隔离原则的方法：将接口B的方法进行拆分细化，将oper1与oper2保留在接口B中，其余的方法重新定义在新的接口C中，如果A需要补充C中的方法，将A实现C的接口就可以。

补充: 接口能够多实现implement，类与类之间只能单继承extend
